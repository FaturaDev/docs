FaturaDev entegrasyonuna başlamadan önce ürün sınırını ve kullanacağınız
public API sözleşmesini doğrulayın. Bir özel entegratör bağlantısı
kullanacaksanız müşterinizin yetkilendirmesini de bu değerlendirmeye ekleyin.

## 1. Ürün modelini anlayın

[Ürün yaklaşımı](/getting-started/key-features) sayfasından ortak
entegrasyon yüzeyinin amacını inceleyin. Ardından
[e-Belge Gateway](/getting-started/core-concepts/e-document-gateway) kavramıyla
FaturaDev'in istek yolundaki yerini doğrulayın.

## 2. Sorumluluk sınırını doğrulayın

[Sorumluluk modeli](/getting-started/core-concepts/responsibility-model),
FaturaDev ile müşterinin yetkilendirdiği özel entegratörün görevlerini ayırır.
Mali imza, GİB iletimi ve özel entegratör hesabı FaturaDev'e taşınmaz.

## 3. Yalnız yayımlanmış sözleşmeyi kullanın

Bir API operasyonunu ancak bu sitedeki kanonik API referansında yer aldığında
kullanılabilir kabul edin. Henüz yayımlanmamış endpoint, payload, kimlik
doğrulama yöntemi, kota veya sağlayıcı kabiliyeti hakkında varsayım yapmayın.
